C# Class

A class in C# follows basic Object Oriented Programming principles. Unlike C++, C# has a garbage collector which automatically manages memory, meaning you don't need to manually delete objects when you are done with them. This makes memory management much safer and easier to work with.

This is an example of a class in C# that has public/private variables and methods 


                    public class Goblin
                    {
                        // Public variables - can be accessed directly from outside the class
                        public float Health = 100;  // Every goblin starts with 100 health
                        public int Damage = 10;

                        // Private variable - only this class can see and modify it
                        private string item;

                        // Since the goblin calculates health inside the class,
                        // the rest of the system doesn't need to see this method
                        private void CalculateHealth(int damage)
                        {
                            Health = Health - damage;
                        }

                        // Default constructor - takes no parameters and uses default values
                        public Goblin()
                        {
                            Damage = 10;
                            Health = 100;
                        }

                        // Overloaded constructor - called when you want to create a goblin
                        // with a custom damage value
                        public Goblin(int damage)
                        {
                            // 'this.Damage' refers to the class variable,
                            // not the parameter being passed in
                            this.Damage = damage;
                        }

                        // In C# there is no destructor like C++, the garbage collector
                        // handles cleanup automatically. However you can implement IDisposable
                        // if you need custom cleanup logic for things like file handles.

                        // Setter - allows you to change the private item variable
                        public void SetItem(string item)
                        {
                            this.item = item;
                        }

                        // Getter - allows outside code to read the private item variable
                        public string GetItem()
                        {
                            return item;
                        }

                        // C# also supports properties which combine getters and setters cleanly
                        public string Item
                        {
                            get { return item; }
                            set { item = value; }
                        }

                        // Public method the rest of the system can call
                        public void ReceiveDamage(int damage)
                        {
                            Console.WriteLine("Goblin took damage");
                            CalculateHealth(damage);
                        }
                    }

In this example we use the Goblin class by creating an object and using its methods. Unlike C++, we do not need to manually delete the object. The C# garbage collector will automatically clean up the object from memory when it is no longer being used. 


                    static void Main(string[] args)
                    {
                        // Create a goblin object - no 'new' pointer needed like in C++
                        // C# handles memory allocation automatically
                        Goblin bob = new Goblin(10);

                        bob.ReceiveDamage(25);  // Bob takes 25 damage
                        bob.ReceiveDamage(100); // Bob takes 100 damage

                        // Check if the goblin is dead
                        if (bob.Health <= 0)
                        {
                            Console.WriteLine("Goblin is dead!");
                            // No need to delete bob - the garbage collector handles this
                        }
                    }

C# also supports a cleaner property syntax that replaces the need for separate getter and setter methods. This is one of the key differences from C++. 


                    public class Goblin
                    {
                        // C# auto-properties - the compiler generates the backing field for you
                        public float Health { get; private set; } = 100;
                        public int Damage { get; set; } = 10;

                        // Read-only property - can only be set in the constructor
                        public string Name { get; }

                        public Goblin(string name, int damage)
                        {
                            this.Name   = name;
                            this.Damage = damage;
                        }

                        public void ReceiveDamage(int damage)
                        {
                            Health -= damage;
                            Console.WriteLine($"{Name} took {damage} damage. Health: {Health}");
                        }
                    }

                    // Usage
                    Goblin bob = new Goblin("Bob", 10);
                    bob.ReceiveDamage(25);
                    // Output: Bob took 25 damage. Health: 75
